# Contributing to PRGate

Thanks for your interest in PRGate.

The best contributions right now are:

- bug reports with clear reproduction details
- feature requests with concrete use cases
- improvements to public docs and contributor guidance
- focused pull requests that stay within the scope of a discussed issue

## Before You Start

- Check existing issues before opening a new one.
- Use the issue templates when reporting bugs or requesting features.
- For security issues, do not open a public issue. Follow `SECURITY.md` instead.

## Pull Requests

Please keep pull requests:

- small enough to review in one pass
- clearly described in terms of problem, approach, and impact
- limited to one logical change when possible

When opening a pull request:

- explain why the change is needed
- note any follow-up work that is intentionally out of scope
- include validation notes for any behavior you changed

## Early Project Expectations

Maintainers may close or redirect issues and pull requests that:

- do not align with the current project direction
- introduce large design changes without prior discussion
- add implementation commitments the project has not adopted

If you are planning a larger change, open a feature request first.
