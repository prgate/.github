# Security Policy

## Reporting a Vulnerability

Please do not disclose security issues in public issues, pull requests, or discussions.

If private vulnerability reporting is enabled in the affected repository, use that channel first.

If no repository-specific private reporting channel is listed yet, report vulnerabilities by email to `k@kazakov.xyz`. Do not include exploit details, secrets, or proof-of-concept payloads in public issues.

## What To Include Privately

When reporting a vulnerability through a private channel, include:

- the affected repository and component
- a short description of the issue
- reproduction steps or conditions
- impact assessment if known
- any suggested mitigation or fix

## Scope

This policy applies to public PRGate repositories and shared GitHub configuration maintained through this organization.
