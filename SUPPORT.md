# Support

## Where To Ask For Help

Use the project issue templates for:

- bug reports
- feature requests
- documentation improvements
- repository-level questions about public PRGate project setup

## Before Opening An Issue

- check whether the question is already answered in repository docs
- search for existing issues that already describe the same problem or request
- keep the report focused on one problem or proposal

## Security Issues

Do not use public issues for vulnerability reports. Follow `SECURITY.md` instead.

## Response Expectations

Response times may vary while the public repositories and maintainer workflows are still being set up.
